# Outcome品質基準

## ノード設計の鉄則（2026-04-03の教訓から）

### 可変スロットのデフォルト値は必ずロスターの語を使う
- ❌ NG: 「忘れられた 柱」（柱はロスター外）→ プレイヤーが入れ替えると手札に入る → 他ノードでoutcomeがない → 歪み
- ✅ OK: 「堅い 氷」（堅い=mod_hard, 氷=subj_ice、両方ロスター内）
- 不変スロット（locked: true）はロスター外でよい（橋、祠等）

### outcome生成はロスターの語のみ
- ❌ NG: ロスター外の語（柱, 神像, 荊, 衛兵等）に対してoutcomeを生成する
- ✅ OK: ロスターの50枚の組み合わせのみ生成する
- 「+1 orig」のような余計なパターンを生成プロンプトに含めない

### 固定スロット制御は最初に実装する
- ノード定義を作る前に `locked: true` が機能することを確認する
- 固定スロットが動かないことは、outcome数制御の前提条件

### 格の歪み判定は前回セッションの確定分類に厳密に従う
- 述語17個の格分類（を格7、に格7、なし3）は2026-04-02セッションで確定済み
- を格ノードでは「に格動詞+なし動詞」が全て歪み。曖昧な判断をしない
- 「待っている」「溶けている」はに格。「街道を待っている」「街道を溶けている」は歪み

### 未実装機能のスロットを配置しない
- evaluate.tsに処理がない機能（副詞の後付け等）のスロットをUI上に出さない
- 空スロット（word: null）が意味不明な空白として表示される

## 文体ルール

### 必須
- **三人称現在形**を基調とした寓話体。状態描写は「〜している」「〜だ」
- **完了相「〜した」は許容**（2026-04-14改訂）。状態描写ではなく、その場で起きた出来事を描く際は自然。「肩が脱臼した」「王が目を逸らした」等の直前の出来事描写はOK。明らかに過去の回想になっているものだけ避ける
- 1 outcome = **1〜3文が目安**。短い方がよいが、4文でも必然性があればOK（体言止めの短文連打等）。冗長でないことが基準
- 乾いたトーン。感情の押し売りをしない
- メタ的な文法解説は禁止（「文法が崩れた」→ NG。「文が歪んだ」→ OK、ただし歪みはdefaultOutcomeの仕事）

### 禁止
- 「あなた」「プレイヤー」等のメタ呼称
- 感情を直接説明する描写（「恐ろしいと感じた」→ NG）。ただし「気がした」は文脈次第で許容（「歓迎されている気がした——いや、風が吹いただけだ」→ OK）
- 同じ文型パターンの3回以上連続使用（「Xが道をYしている。Zだ。」の繰り返し等）

## 存在シート準拠

- 各outcomeは `existence-sheets.md` の **interpretation** または **alt** に基づく
- interpretationが第一候補。altはノードの文脈に合う場合のみ使用
- 存在シートに書かれていない独自解釈は**原則禁止**。どうしても必要なら🚩フラグ
- 同じ存在が複数ノードに出る場合、解釈の**一貫性**を保つ
  - 例: 「壊れた子猫」は常にぬいぐるみ（interpretationに従う）
  - ただしaltの使い分けヒントに合致するなら、ノードごとに別のaltを使ってよい

## 格の整合性

### 格歪みルール（2026-04-04改訂）

| 目的語の格 | 歪みとなる述語 | 歪み数 |
|-----------|--------------|-------|
| を格 | 眠っている, 絡みついている, 降り注いでいる, 囁いている, 溶けている, 歌っている, 踊っている, 笑っている | 8/17 |
| に格 | 塞いでいる, 守っている, 追いかけてくる, 隠している, 砕く, 見つめている | 6/17 |
| から格 | 塞いでいる, 守っている, 追いかけてくる, 隠している, 砕く, 眠っている, 待っている, 絡みついている | 8/17 |
| で格 | 塞いでいる, 守っている, 追いかけてくる, 隠している, 砕く, 絡みついている, 降り注いでいる | 7/17 |

- 「逃げていく」は経路のを格として常に許容
- 「見つめている」は「を/から」格動詞。に格では「Xに見つめている」は非文なので歪み
- 「待っている」はを格でも許容（「バスを待つ」「鐘を待つ」等）。2026-04-08改訂: を格歪みリストから除外
- 「眠っている/絡みついている」はに格動詞。から格では「Xから眠っている」等は非文なので歪み
- 「輝く」はを格でも許容（「夜空を輝く星」のような詩的用法）。2026-04-17改訂: を格歪みリストから除外

## damage / quill 基準

### HP 15 環境下での注意（2026-04-06改訂）
- HP 20→15 に減少。damage 5 = HPの1/3。2回で瀕死
- 全体AP 12 = 全ノードで最適解を選べない。ノーチェンジ通過（初期値のdamageを食らう）が戦略上必要になる
- ノーチェンジ通過のリスクとAP温存のトレードオフがゲームの核
- 初期値のoutcomeは「そこそこ痛いが致命的ではない」レベル（damage 2-3）が理想

### damageの目安
| 状況 | damage |
|------|--------|
| 完全に無害・有益 | 0 または -1（回復）、稀に -2（大きな回復。篝火・泉等の安全な場面のみ） |
| 軽い被害 | 1-2 |
| 初期値そのまま（改善なし） | 2-3 |
| 有害だが通過可能 | 3-4 |
| 致命的な脅威 | 5 |
| 歪み（defaultOutcome） | 4-5 |

### quillの目安
| 状況 | quill |
|------|-------|
| 歪み | 0 |
| 改善なし・ほぼ変化なし | 1-2 |
| まあまあの書き換え | 3-4 |
| うまい書き換え | 5-6 |
| 見事な書き換え | 7-8 |
| 傑作（稀に） | 9-10 |

### 判断基準
- **存在の脅威度**が高い（竜、炎、鋭い等）→ damage高め、quill低め
- **存在の脅威度**が低い（子猫、小さな、優しい等）→ damage低め、quill高め
- **意外性のある組み合わせ**が面白い結果を生む → quillボーナス
- **修飾語が主語の性質を強める**（燃え盛る炎、堅い岩）→ 効果が極端に
- **修飾語が主語の性質を裏切る**（壊れた竜、優しい嵐）→ 意外性でquill高め

## rewardItems / rewardCards

- **全outcomeの5-15%** 程度にrewardItemsを付与（ノードによって差があってよい。付与0%のノードは避ける）
- 高quill（6以上）のoutcomeに付ける傾向
- アイテム名と説明は存在の解釈に紐づく（「壊れた竜」→ 「折れた竜牙」等）
- sellPrice: 5-20の範囲。quillが高いoutcomeほど高め
- rewardCardsは**非常に稀**（全体の2-3%以下）。特別な組み合わせのみ

## セルフレビューチェックリスト

生成後、以下を確認:
1. ☐ 同じ文型パターンが3回以上連続していないか
2. ☐ 存在シートの解釈から逸脱していないか
3. ☐ 格の整合性（自動詞+を格→歪み等）
4. ☐ damage/quillが基準の範囲内か
5. ☐ 修飾語の効果が反映されているか（小さな→無害化、燃え盛る→脅威増大）
6. ☐ 同じ主語の異なる修飾語outcomeで変化があるか（全部同じ展開になっていないか）
7. ☐ テキストに「面白さ」があるか（テンプレ臭がしないか）
8. ☐ rewardItemsの割合が5-15%か（0%は避ける）

## 🚩フラグ基準

以下の場合、outcome横に🚩を付けて `review-needed.md` に記載:
- 存在シートにない独自解釈を使った
- damage/quillの判断が微妙（面白いけど強すぎる等）
- 同じ主語で似た展開になってしまったが良い代替が思いつかない
- テキストの面白さに自信がない
- 修飾語の効果が薄い（修飾語を変えても展開が同じ）
