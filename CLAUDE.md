# Word Mage（言葉の魔法使い）

## コンセプト
Slay the Spire的ノード進行 × 自然言語書き換え × 手札管理パズルゲーム。
プレイヤーは「校正者（リヴァイザー）」として、文章のスロット（主語・述語・修飾語等）を手札の語と入れ替え、世界を書き換えながら進む。

## 技術スタック
- **フレームワーク**: Svelte 5 + TypeScript
- **ビルド**: Vite（base: '/word-mage/'）
- **データ**: 静的TypeScript（ノード定義、語彙、タグはnodes.ts内）
- **描画**: HTML/CSS（古い本の見開きUI）
- **デプロイ**: GitHub Pages（gh-pagesブランチ、`npx gh-pages -d dist`で更新）
- **将来**: Tauri でデスクトップ/Steam対応

## ゲーム設計

### ノードタイプ
| タイプ | 構造 | 説明 |
|--------|------|------|
| puzzle | 1文 | 環境/危機を書き換えて突破 |
| elite | 1文 | 高難度・高報酬 |
| rest | 1文 | 泉。書き換えなければHP回復、書き換えると面白いイベント |
| battle/boss | 2文・複数ターン | 敵行動文+プレイヤー行動文。敵HPあり |
| shop | 専用UI | 語カード購入・手札売却・アイテム売却 |
| treasure | 専用UI | カードプールからランダムで語カード獲得 |

### 1文構成ノード（puzzle/elite/rest）
- スロット最大4（読みやすさのため）
- 助詞はスロットカテゴリから自動生成（subject→が、object→を、object_ni→に等）
- 判定: Outcome配列を先頭から走査、最初にマッチした結果を適用
- マッチしない場合 → defaultOutcome（文の歪み: 高ダメージ、ゴールド0）
- 「書き換えなかった場合」も明示的にOutcomeとして定義

### 2文構成バトル（battle/boss）
- 敵行動文: 毎ターン新しい攻撃パターン（ローテーション）
- プレイヤー行動文: 毎ターンリセット。手札で強化して敵にダメージ
- APは両文で共有 → 攻防のAP配分がStSの攻撃/ブロック振り分けに相当
- 敵HPが0になれば勝利

### 判定方式: 組み合わせ定義ベース
- 各Outcomeに`conditions`（スロットID→必要タグのAND条件）
- 配置順: 複合条件（具体的）→ 単体条件 → 元テキスト条件
- defaultOutcome = 「文の歪み」（未定義の組み合わせ = 文法破綻とみなす）
- LLMで組み合わせパターンとテキストを大量生成 → 人間がキュレーション

### リソース
- **HP 15/15**: ダメージ/回復。0でゲームオーバー（2026-04-06改訂: 20→15）
- **AP 12（面ごとリセット）**: スロット1回入れ替え=AP 1消費。各面でpuzzle 4-5個にAPを配分。面ごとにリセット
- **Quill**: 書き換えの腕前が筆に宿る力。ショップで消費
- **手札**: 初期5枚（mod×2, subj×1, pred×1, adv×1、ランダム選出）、上限7（永続カードで+2可）。同カテゴリのスロットにのみ入る
- **アイテム**: 換金アイテム（ショップで売れる）。うまい書き換えで入手
- **永続カード**: 手札枠を使ってパッシブ効果（AP+1、手札上限+2、毎ノード+2Q）

### マップ（2026-04-08改訂: 5面構成）

- **スタートノード（層0）は自動通過**（イベントなし、nodeDefId='__start__'）
- NODE_POOLは36個（puzzle23 + rest7 + elite6）。5面で計27個使用（puzzle21 + rest3 + elite3）。puzzle消費率91%

### 世界観・トーン
- 文体: 乾いた寓話体、三人称現在形
- 古い童話のような質感。ビジュアルは「古い本の見開き」だが、世界＝書物という設定を文字通りに語るわけではない
- 物語性はフレーバー程度。重い背景設定は置かない
- **根底の哲学**: 言葉を通して世界を書き換える（認識・解釈する）力は人類の文化の根本であり、それこそが最大の魔法である。聖書も歴史も言葉によって作られてきた。そこに魔法的な力を見る

### 主人公・旅の動機
- **主人公**: 若い魔女の女の子。ひとり旅をしている
- 幼児的な全能感で、世界の書き換えを素朴に楽しんでいる
- 旅の動機は軽い。なぜ言葉で世界を変えられるかは説明しない——魔女だから
- 冒頭の一文: **「小さな 魔女が 歩いている」**

### ゴール・エンディング（2026-04-03決定）
- 旅のゴール: **泉**。旅の果てに泉がある
- 魔女が泉を覗き込むと、水面に自分の姿が一文として映る
- その一文の**修飾語＋主語（＝存在）スロットが開き**、手札から書き換えられる
- 書き換えた存在として生きる**短いエピローグ**が流れて終わる
- カードが残っていなければ「小さな 魔女」のまま。主語だけなら「小さな 竜」等もOK
- 187存在に対応するエピローグを手書き（存在シートの延長）
- **構造的意味**: 言葉で世界を書き換えてきた子が、最後に自分自身を書き換える——幼児的全能感の帰結
- 周回ごとにビルドが違う → 最後の存在が変わる → 違うエピローグ → 周回動機

### 選定基準

#### 主語・修飾語
1. **方向性テスト**: 「Xが道を塞いでいる」で効果の方向がわかるか
2. **ユーモアテスト**: 「屈強なX」で笑えるか。修飾語との化学反応
3. **述語レンジテスト**: 塞ぐ/守る/逃げる/砕くで全部違う絵が浮かぶか
4. 存在そのものが効果の方向性を帯びているか

#### 述語
1. **両面性テスト**: 存在次第で有益/有害が反転するか
2. **存在レンジテスト**: 17主語のどれでも絵が浮かぶか
3. **場面テスト**: 冒険のどんなシーンを生むか
4. **差別化テスト**: 他の述語と同じ展開にならないか

### 副詞 5（確定）
| 副詞 | 方向 | damage補正 | quill補正 |
|------|------|-----------|----------|
| 静かに | 弱化・穏和 | -1 | 0 |
| 激しく | 強化・過激 | +1 | +1 |
| ゆっくり | 減速・猶予 | -1 | 0 |
| 力なく | 弱体・衰退 | -1 | 0 |
| 突然 | 急変・不意 | 0 | +2 |

- **程度系のみ**をプレイヤーカードとする
- 変質系（逆さまに/繰り返し/楽しそうに等）はノード固定の不変要素としてのみ使用
- **副詞スロットは重要ノード（3スロット）でのみ使用**。通常ノードには置かない
- 処理方式（2026-04-03改訂）:
  - **通常の2スロットノード**: 副詞スロットなし。副詞カードは使えない
  - **3スロットノード（重要ノード）**: 副詞ごとに固有テキストを手書き
  - **後付け定型句方式は不採用**: 「——静かに、音もなく。」等の固定サフィックスはoutcome本文の文脈とミスマッチする。テキストに影響させるなら固有テキストを書く
  - **暫定**: evaluate.tsにdamage/quill数値補正は実装済み（副詞固有テキスト未実装ノード向け）

- 夢(主語): 方向性テスト不合格。修飾語依存の空の器
- 花(主語): 修飾語との化学反応が薄い
- 老人(主語): 方向性弱い
- 鬼/盗賊(主語): 述語レンジ狭い
- 狼(主語): 竜と被る
- 大きな(修飾語): ユーモア弱い → 屈強な に差し替え
- 密かに(副詞): 静かにと近い
- 悲しそうに(副詞): 力なくと方向性が近い
- 逆さまに/繰り返し/楽しそうに(副詞): 変質系のため後付け不可。ノード固定でのみ使用

### 後から追加する場合のコスト
- 修飾語1枚追加: 存在シート17枚 + 気になるオーバーライド数個
- 主語1枚追加: 存在シート11枚 + 気になるオーバーライド数個
- 述語1枚追加: 述語可変ノードのoutcomeに追加。述語固定ノードは影響なし
- 初期リリース時に完璧でなくてよい。遊んでから足す

## Outcome設計

### 現行方式（Web簡易版: 2026-04-02実装済み）
- 全有効組み合わせに固有テキストを1つずつ記述（カードID直接指定）
- conditions: `{ s1: 'h1', s2: 'shop_3', s4: 'shop_4' }` （string=カードID, string[]=タグ互換）
- 格歪みルール（2026-04-08改訂）: を格歪み9種 / に格歪み6種 / から格歪み8種 / で格歪み7種。詳細は `quality-criteria.md` の格歪みテーブル参照
- 逃げていくは経路のを格として常に許容
- 待っているはを格でも許容（「鐘を待つ」等）
- 歪みテキストは世界観内で書く（メタ的文法解説はNG）
- 全6通常ノード+ボス4判定文 = 1,156 outcomes 完了

### スケーリング方針（2026-04-02 設計決定・同日改訂）

#### 核心思想: 「存在」が面白さの中心
- **修飾語×主語 = 「存在」**（例: 壊れた子猫, 燃え盛る竜）がこのゲームの楽しさの核
- 存在の「創造的解釈」（壊れた子猫→ぬいぐるみ）が全ノードで再利用される

#### テンプレート方式は却下
- node_1で実験: 3段階分岐テンプレートを試作 → テキストの面白さが死ぬ
- 「ぬいぐるみだった」「長い交渉の末」等の固有表現がテンプレートで均される
- 効果（damage/quill）の乖離も28通り中8通りで発生
- **結論: テキストは手書き。存在シートは手書きの参照資料として使う**

#### スロット数デザインルール（2026-04-03決定）

| 場面 | 可変スロット数 | 理由 |
|------|-------------|------|
| 通常ノード | **2** | 基本。認知負荷を抑える |
| 重要ノード（中ボス前等） | **3** | 副詞追加で特別感。ここぞの場面のみ |
| 通常ボス戦 | **2 × 2文** | 敵文+自文で既に考えることが倍。スロット数は抑える |
| ラスボス | **3 × 2文もアリ** | クライマックスの特別感 |

- 通常ノードは2スロットが基本。プレイヤーが1手で文の意味を大きく変える体験
- 3スロットは組み合わせ爆発を起こすため、特別な場面に限定する
- 副詞スロットは3スロット目として重要ノードでのみ使う。その場合、副詞ごとに固有テキストを書く（後付け定型句は文脈とミスマッチするため不採用）

#### 不変スロットによるノード個性化 + 組み合わせ数制御
- ノードごとに「不変要素」を設定し、変更可能なスロットを制限する
- 不変要素がノードの個性・パズルの制約を生み、同時に組み合わせ数を管理可能にする
- 不変スロットは `locked: true` で実装。ドラッグ・抜き取り・挿入全て不可
- **不変スロットの語はロスター外でよい**（橋、祠、街道、泉、玉座等のノード固有語）
- **可変スロットのデフォルト値は必ずロスターの語を使う**（プレイヤーが入れ替えた時に手札に入るため）

| パターン | 可変スロット | 組み合わせ数/ノード |
|---------|------------|-------------------|
| 存在のみ | mod+subj | 11×17 = 187 |
| 主語+述語 | subj+pred | 17×17 = 289（歪み除外後はそれ以下） |
| 修飾語+述語 | mod+pred | 11×17 = 187（歪み除外後はそれ以下） |
| 主語のみ | subj | 17 |
| 述語のみ | pred | 17 |

ノード骨格（2026-04-03実装済み）:
```
A: 忘れられた橋   [mod*][subj*]が 橋を 塞いでいる       ← 存在のみ可変(187)
B: 沈む祠        [mod*][subj*]が 祠に 眠っている       ← 存在のみ可変(187)
C: 霧の街道      呪われた[subj*]が 街道を [pred*]       ← subj+pred(119, 歪み180)
D: 泉の声        [mod*]歌が 泉から [pred*]            ← mod+pred(132, 歪み60)
E: 王の間        [mod*][subj*]が 玉座を 守っている      ← 存在のみ可変(187)
```

#### 存在シート
- interpretation（デフォルトの絵）+ alt（文脈別の読み替え、ヒント付き）
- 創造的解釈が先、特性体系は後（解釈を書いてから帰納的に分岐軸を見つける）
- 187存在の解釈作成済み: `src/game/data/existence-sheets.md`
- 特性体系は未着手（187存在の解釈から帰納的に導出予定）

#### 制作方式
- 全outcome手書き + 存在シートを参照しながら書く
- LLMワークフロー: Sonnet下書き→Opus磨き上げ（前回セッションで確認済み）
- 副詞は後付け方式（テキスト本体を増やさない）
