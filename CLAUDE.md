# Word Mage（言葉の魔法使い）

## コンセプト
Slay the Spire的ノード進行 × 自然言語書き換え × 手札管理パズル。
プレイヤーは「校正者（リヴァイザー）」として、文章のスロット（主語・述語・修飾語等）を手札の語と入れ替え、世界を書き換えながら進む。

## 技術スタック
- **フレームワーク**: Svelte 5 + TypeScript
- **ビルド**: Vite
- **データ**: 静的JSON（ノード定義、語彙辞書、タグ定義）
- **描画**: HTML/CSS + Canvas（背景エフェクト）
- **音声**: Web Audio API
- **将来**: Tauri でデスクトップ/Steam対応

## ディレクトリ構成
```
src/
  components/    # Svelteコンポーネント（UI部品）
  game/
    data/        # ノード定義JSON、語彙辞書JSON、タグ定義
    engine/      # ゲームロジック（判定エンジン、手札管理、状態管理）
  styles/        # グローバルCSS、テクスチャ用スタイル
  assets/
    sounds/      # SE素材
    textures/    # 紙質感などのテクスチャ画像
```

## ゲーム設計の要点

### コアメカニクス
- 各ノードのイベントは構文スロットに分解された文章
- スロット種別: 主語(◆青) / 述語(●赤) / 対象語(▲緑) / 修飾語(■黄) / 副詞(★紫)
- 操作: 入れ替え(Swap) / 抜き取り(Extract) / 挿し込み(Insert) / 消去(Erase)
- 判定: タグベースの静的システム（LLMランタイム依存なし）

### 多解性
- 同じノードを複数の手札構成で突破可能
- 脅威スコア = 主語の脅威度 × 述語の攻撃性 × 修飾語の強度
- アクロバット解（意外な組み合わせ）にボーナス

### 画面設計
- 全画面が「古い童話の本の見開き」
- 左ページ: 現在のイベント文（操作エリア）
- 右ページ: 書き換え結果のテキスト + 属性エフェクト
- 本の下（机上）: 手札エリア
- ページめくりアニメーションでノード遷移
- 固定アスペクト比（16:9）、レターボックス

### 質感・演出
- ブラウザ臭を徹底排除（カスタムカーソル、テキスト選択無効、右クリック無効）
- 背景Canvas層で紙の質感、環境光、パーティクル
- サウンド二層構造: 操作時はアナログ質感（紙・インク）、発動時は魔法的SE
- タグベースの抽象エフェクト（火→赤い光、水→青い波紋など）
- 個別イベントCGは用意しない。テキスト + 属性エフェクトで完結

### 世界観・トーン
- 「書かれた世界」：世界は一冊の書物として存在
- 文体: 乾いた寓話体、三人称現在形、20〜40字/文
- 物語性はフレーバー程度。ドラマや感情アークは不要
- 「魔法＝言葉のハック」という哲学が通底する
- 童話のような古い物語の質感

## 開発方針
- プロトタイプ: 5〜7ノード、手札5枚、語句20〜30種、プレイ時間10〜15分
- Web簡易版を公開 → 製品版はTauriでSteam販売
- LLMはコンテンツ生成パイプライン（開発時）に活用、ランタイムでは使わない
